# <a name="manage-gdpr-data-subject-requests-with-the-dsr-case-tool-in-the-microsoft-365-compliance-center"></a>Управление запросами субъектов данных, регулируемых GDPR, с помощью средства управления делами DSR в Центре соответствия требованиям Microsoft 365

Общий регламент по защите данных (GDPR) Европейского союза предназначен для защиты и реализации прав физических лиц на конфиденциальность внутри Европейского союза (ЕС). GDPR предоставляет физическим лицам в Европейском Союзе (известным как субъекты данных) право на доступ, извлечение, исправление, удаление и ограничение обработки их персональных данных. Согласно GDPR, личные данные представляют собой любую информацию, относящуюся к физическому лицу, личность которого установлена или может быть установлена. Официальный запрос физического лица к организации, требующий выполнения некоторых действий с персональными данными этого лица, называется запросом субъектов данных, или DSR. Подробные сведения об ответе на запросы субъектов данных для данных в Office 365 см. в [Руководстве по запросам субъектов данных в Office 365](https://go.microsoft.com/fwlink/?linkid=871169 ).
  
Для управления расследованиями в ответ на запрос DSR, отправленный сотрудником вашей организации, вы можете использовать средство управления делами DSR в Центре соответствия требованиям для поиска содержимого, хранящегося в перечисленных ниже местах.
  
- Любой почтовый ящик в вашей организации. Сюда же входят беседы в Skype для бизнеса и приватные чаты в Microsoft Teams

- Все почтовые ящики, связанные с группой Microsoft 365, и все почтовые ящики команды в Microsoft Teams

- Все сайты SharePoint Online и учетные записи OneDrive для бизнеса, принадлежащие вашей организации.

- Все сайты Microsoft Teams и сайты групп Microsoft 365 принадлежащие вашей организации

- Все общедоступные папки в Exchange Online.

С помощью средства DSR можно:
  
- Создавать отдельные дела для каждого расследования DSR.

- Управлять доступом различных пользователей к делу DSR, добавляя их в качестве участников дела. Только участники дела могут получать доступ к делу. В списке дел на странице **Дела запросов субъектов данных** в Центре соответствия требованиям они могут видеть только те дела, участниками которых являются. Кроме того, вы можете назначать различные разрешения различным участникам одного и того же дела. Например, вы можете разрешить одним участникам только просматривать дело и результаты поиска, а другим — создавать операции поиска и экспортировать результаты поиска.

- Применять встроенные средства для поиска всего контента, связанного с определенным субъектом данных.

- При необходимости изменять поисковый запрос, используемый по умолчанию, и повторно выполнять поиск, чтобы сузить количество результатов поиска.

- Добавлять другие поисковые запросы контента, связанные с делом DSR. Сюда входит создание поисковых запросов, возвращающих частично индексированные элементы и системные журналы из службы роуминга Office.

- Экспортировать данные в ответ на запрос DSR на доступ или экспорт.

- Удалить дело по завершении процесса расследования для ответа на запрос субъекта данных. В результате будут удалены все операции поиска контента и экспорта, связанные с данным делом.

> [!IMPORTANT]
> Наши инструменты помогают администраторам выполнять запросы DSR и экспортировать их, используя встроенные функции поиска и экспорта в средстве управления делами DSR. Это средство помогает упростить оптимальный способ экспорта данных, релевантных для запроса DSR, отправленного субъектом данных. Однако важно отметить, что результаты поиска могут различаться в зависимости от субъекта данных и действий администратора: от них зависит, будет ли элемент считаться "персональными данными" для целей экспорта. Например, если субъект данных последним редактировал файл, созданный не им, возможно, этот файл не будет возвращен в результатах поиска. Аналогичным образом, администратор может экспортировать данные, не включая частично индексированные элементы или все версии документов SharePoint. Следовательно, предоставленные средства могут помочь упростить доступ к запросам данных и их экспорту; однако результаты зависят от конкретного сценария использования средства администраторами и субъектами данных.
  
## <a name="step-1-assign-ediscovery-permissions-to-potential-case-members"></a>Шаг 1. Назначьте разрешения на обнаружение электронных данных потенциальным участникам дела

По умолчанию глобальный администратор имеет доступ к средству управления делами DSR в Центре соответствия требованиям Microsoft 365. Система преднамеренно спроектирована так, что другие пользователи, такие как сотрудник по конфиденциальности данных, руководитель отдела кадров или другие люди, участвующие в расследованиях DSR, не имеют доступа к средству управления делами DSR. Чтобы эти пользователи получили доступ, им должны быть назначены соответствующие разрешения. Для этого проще всего добавить пользователей в группу обнаружения электронных данных на странице **Разрешения** в Центре соответствия требованиям. Кроме того, необходимо назначить эти разрешения, чтобы добавить пользователей в качестве участников дела DSR, созданного на шаге 2. 
  
Пошаговые инструкции см. в статье [Назначение разрешений для обнаружения электронных данных](/microsoft-365/compliance/assign-ediscovery-permissions).
  
> [!NOTE]
> По умолчанию у глобального администратора (и других участников группы ролей "Управление организацией" в Центре соответствия требованиям) нет необходимых разрешений для экспорта результатов поиска контента (см. шаг 4 в данной статье). Чтобы решить эту проблему, администратор может добавить сам себя в качестве участника группы ролей "Диспетчер обнаружения электронных данных". 
  
## <a name="step-2-create-a-dsr-case-and-add-members"></a>Шаг 2. Создайте новое дело DSR и добавьте участников

Следующий этап — создание дела DSR. При создании дела можно запустить встроенный поиск или создать дело, не запуская поиск. В следующей процедуре показано, как создать дело, не запуская поиск, а затем добавить участников в дело.
  
1. Перейдите на страницу <https://compliance.microsoft.com> и выполните вход со своей рабочей или учебной учетной записью.

2. В расположенной слева области Центра соответствия требованиям Microsoft 365 выберите **Запросы субъектов данных**.

3. На странице **Запросы субъектов данных** щелкните **Создать дело**.

4. На странице мастера **Новое дело DSR** введите название дела, при желании добавьте описание, а затем нажмите **Далее**. Название дела должно быть уникальным в пределах организации.

    > [!TIP]
    > Рассмотрите возможность добавления имени человека, отправившего запрос DSR, который вы расследуете, в название и/или описание нового дела. Обратите внимание, что только участники этого дела (и администраторы обнаружения электронных данных) смогут видеть дело в списке дел на странице **Запросы субъектов данных**.
  
5. На странице **Детали запроса** в области **Субъект данных (лицо, создавшее этот запрос)** выберите человека, для которого вы хотите найти и экспортировать данные, а затем нажмите **Далее**.

6. На странице **Подтвердите параметры дела** можно изменить имя или описание дела, а также выбрать другого субъекта данных. Если это не нужно, нажмите кнопку **Далее**.

    Откроется страница с подтверждением, что дело DSR создано.

    ![Запустите поиск или закройте страницу нового дела DSR](../media/b5e62c2c-cafe-4a8d-a38c-789ed9f9ccbd.png)
  
    В этот момент можно выполнить одно из следующих действий:

    а. Щелкнуть **Показать результаты поиска** и запустить поиск. Этот вариант выбран по умолчанию. Встроенный поиск, который запускается при выборе этого варианта, и возвращаемые результаты будут рассмотрены на шаге 3.

    б. Нажать **Готово**, чтобы закрыть окно создания дела DSR без запуска встроенного поиска. При выборе этого варианта новое дело DSR будет показано на странице **Запросы субъектов данных**.

7. Нажмите **Готово**, чтобы перейти в новое дело DSR и добавить в него участников.

8. На странице **Запросы субъектов данных** щелкните имя только что созданного дела DSR.

9. Во всплывающем окне **Управление делом** в разделе **Управление участниками** нажмите **Добавить**.

    Появится список **Пользователи**, для которых назначены соответствующие разрешения на обнаружение электронных данных. В этом списке перечислены люди, которым вы на шаге 1 дали разрешения на обнаружение электронных данных.

10. Выберите людей, которых нужно добавить в качестве участников дела DSR, нажмите **Добавить** и сохраните изменения.

    Вы также можете добавить группу ролей в дело, нажав кнопку **Добавить** в разделе **Управление группами ролей**. 

## <a name="step-3-run-the-search-query"></a>Шаг 3: Запустите поисковый запрос

После создания дела DSR и добавления участников необходимо запустить встроенный поиск, связанный с делом. Этот стандартный поисковый запрос выполняет следующие действия.
  
- Ищет во всех почтовых ящиках организации сообщения электронной почты, отправленные или полученные субъектом данных. Для этого используется свойство сообщения электронной почты *Участники*, которое ищет субъекта данных во всех полях сообщения, где перечислены люди. Это свойство возвращает элементы, в которых субъект данных находится в полях **Отправитель**, **Адресат**, **Копия** и **Скрытая копия**. Кроме того, поиск сообщений, отправленных или полученных субъектом данных, производится в общедоступных папках Exchange Online.

- Проводится поиск документов и элементов, созданных или загруженных субъектом данных на всех сайтах организации. Эта операция выполняется с помощью следующих свойств сайта:

  - Свойство *Автор* возвращает элементы, в которых субъект данных указан в поле "Автор" документа Office. Это значение сохраняется, даже если документ скопирован и отправлен другим пользователем.

  - Свойство *Создатель* возвращает элементы, созданные или отправленные субъектом данных.

Вот как выглядит запрос по ключевым словам для встроенного поиска, автоматически создаваемого при создании дела DSR.
  
```powershell
participants:"<email address>" OR author:"<display name>" OR createdby:"<display name>"
```

Например, если имя субъекта данных — Инга Леонтьева, запрос по ключевому слову будет выглядеть так:
  
```powershell
participants:"ina@contoso.com" OR author:"Ina Leonte" OR createdby:"Ina Leonte"
```

 **Чтобы запустить встроенный поиск для дела DSR:**
  
1. В Центре соответствия требованиям Microsoft 365 щелкните **Запросы субъектов данных**, выберите дело, созданное на шаге 2, и щелкните **Открыть дело**.

2. Выберите вкладку **Поиск** в верхней части страницы, а затем установите флажок рядом со встроенным поиском, созданным при создании дела DSR. Имя поиска такое же, как у дела DSR. 

3. На всплывающей странице поиска щелкните **Повторить**, чтобы начать поиск.

4. После завершения поиска нажмите кнопку **Предварительный просмотр результатов** для предварительного просмотра результатов поиска. Подробнее см. в разделе [Предварительный просмотр результатов поиска](/microsoft-365/compliance/content-search#preview-search-results).

    > [!TIP]
    > Вы также можете просмотреть статистику поискового запроса, чтобы увидеть количество почтовых ящиков и элементов сайта, найденных при поиске, а также основные расположения контента, в которых находятся элементы, соответствующие поисковому запросу. Подробнее см. в разделе [Просмотр сведений и статистики поиска](/microsoft-365/compliance/content-search#view-information-and-statistics-about-a-search). 
  
Вы можете отредактировать встроенный поисковый запрос, изменить расположения контента, в которых ведется поиск, а затем снова запустить поиск. Дополнительные сведения см. в разделе [Шаг 5](#optional-step-5-revise-the-built-in-search-query).
  
## <a name="step-4-export-the-data"></a>Шаг 4. Экспортируйте данные

Выполнив встроенный поиск, можно экспортировать результаты поиска. Кроме того, перед экспортом данных может потребоваться отредактировать запрос, чтобы сузить результаты поиска. Дополнительные сведения о фильтрации результатов см. в описании шага 5.
  
При экспорте результатов поиска элементы почтовых ящиков можно скачать в виде PST-файлов или в виде отдельных сообщений. При экспорте контента с сайтов SharePoint и из учетных записей OneDrive будут экспортированы копии документов Office и других документов. К результатам поиска прилагается файл результатов, содержащий сведения о каждом экспортируемом элементе. Подробнее об экспорте см. в разделе [Экспорт результатов поиска контента](/microsoft-365/compliance/export-search-results).
  
> [!NOTE]
> По умолчанию у глобального администратора (и других участников группы ролей "Управление организацией" в Центре соответствия требованиям) нет необходимых разрешений для экспорта результатов поиска контента. Чтобы решить эту проблему, администратор может добавить сам себя в качестве участника группы ролей "Диспетчер обнаружения электронных данных". 
  
Компьютер, используемый для экспорта результатов поиска, должен соответствовать указанным ниже требованиям к системе.
  
- 32- или 64-разрядный выпуск Windows версии не ниже 7
    
- Microsoft .NET Framework 4.7
    
- поддерживаемый браузер:
    
  - Microsoft Edge
    
    или
    
  - Internet Explorer версии не ниже 10.
    
    > [!NOTE]
    > Корпорация Майкрософт не производит сторонние расширения и надстройки для приложений ClickOnce. Экспорт данных с помощью неподдерживаемого браузера со сторонними расширениями или надстройками не поддерживается. 
  
 **Чтобы экспортировать данные из встроенного поиска в деле DSR:**
  
1. В Центре соответствия требованиям Microsoft 365 щелкните **Запросы субъектов данных**, выберите дело, созданное на шаге 2, и щелкните **Открыть дело**.

2. Выберите вкладку **Поиск** в верхней части страницы, а затем установите флажок рядом со встроенным поиском, созданным при создании дела DSR.

3. На всплывающей странице поиска выберите **Экспорт результатов**. 

4. На странице **Экспорт результатов** выберите следующие рекомендуемые параметры для запросов на экспорт DSR. 

    ![Настройка параметров экспорта.](../media/25416b79-57da-46a1-ae07-e640602a8fa4.png)
  
    а. В параметре **Варианты вывода** выберите первое значение (**Все элементы, кроме тех, которые имеют неизвестный формат, зашифрованы или не были проиндексированы по другим причинам**), чтобы экспортировать только индексные элементы. Экспортировать частично индексные элементы из встроенного поиска нежелательно, потому что частично индексированные элементы от других пользователей также будут экспортироваться. Чтобы экспортировать только частично индексированные элементы для конкретного субъекта данных, рекомендуется создать отдельный поиск. Подробнее см. статью [Экспорт частично индексированных элементов](#exporting-partially-indexed-items), раздел "Дополнительные сведения об использовании средства управления делами DSR".

    б. Для параметра **Экспортировать содержимое Exchange как** выберите третье значение, **Один PST-файл, содержащий все сообщения в одной папке**. Так как некоторые результаты могут относиться к элементам, отправленным из почтового ящика другого пользователя, при этом значении параметра элемент будет просто перечислен в одной папке без указания фактического почтового ящика. Это лучший вариант, когда вы затем удаляете дубликаты результатов, как рекомендуется сделать далее. Это значение параметра также позволяет субъекту данных просматривать элементы в хронологическом порядке (они сортируются по дате отправки), не переходя каждый раз из одной папки в другую.

    в. Выберите **Включить удаление дубликатов**, чтобы исключить повторяющиеся сообщения электронной почты. Рекомендуется использовать этот вариант, так как встроенный поиск ищет во всех почтовых ящиках организации. Если выбрать это значение параметра, будет экспортироваться единственная копия сообщения, даже если в почтовых ящиках найдено несколько копий. Использование этого значения вместе с экспортом сообщения в один PST-файл в одной папке, удобнее всего для пользователей запроса на экспорт DSR. В отчете об экспорте Results.csv перечислены все расположения, в которых обнаружены дублирующиеся сообщения.

    При желании можно также выбрать параметры **Включить версии для документов SharePoint**, чтобы экспортировать все версии документов SharePoint и OneDrive. Для этого необходимо, чтобы у библиотек документов было включено управление версиями. Этот параметр помогает обеспечить экспорт всех данных, имеющих отношение к делу.

5. Выбрав параметры экспорта, нажмите кнопку **Экспорт**.

    Начнется подготовка результатов поиска для скачивания. Это означает, что они отправляются в хранилище Azure вашей организации в Microsoft Cloud. Далее показано, как скачать эти данные на локальный компьютер.

6. Перейдите на вкладку **Экспорт**, чтобы увидеть на экране созданное вами задание экспорта. Имена заданий экспорта отличаются от имен соответствующих запросов на поиск контента только тем, что в конце добавлено **_Export**.

7. Щелкните только что созданное задание экспорта, чтобы увидеть всплывающую страницу экспорта. На ней показаны сведения о поиске, такие как размер и общее количество экспортируемых элементов, а также процент элементов, переданных в хранилище Azure. Нажмите **Обновить**, чтобы обновить сведения о состоянии отправки. 

8. В разделе **Ключ экспорта** щелкните **Скопировать в буфер обмена**. Вы используете этот ключ для скачивания результатов поиска на шаге 11.

9. Щелкните значок ![экспорта результатов поиска](../media/47205c65-babd-4b3a-bd7b-98dfd92883ba.png) **Скачать результаты** в верхней части страницы экспорта. 

10. Во всплывающем окне в нижней части страницы нажмите кнопку **Открыть**, чтобы открыть **средство экспорта eDiscovery**. **Средство экспорта eDiscovery** будет установлено, когда вы впервые скачаете результаты поиска. 

11. В **средстве экспорта eDiscovery** вставьте в соответствующее поле ключ экспорта, который вы скопировали на шаге 8.

12. Нажмите кнопку **Обзор** и укажите расположение, в которое вы хотите скачать файлы результатов поиска. 

    > [!NOTE]
    > В связи с высокой активностью диска (большим количеством операций чтения и записи) результаты поиска следует скачать на локальный диск; не скачивайте их на сетевой диск или в другое сетевое расположение.

    **Средство экспорта службы обнаружения электронных данных** отображает сведения о состоянии процесса экспорта, в том числе о предполагаемом количестве (и размере) элементов, которые осталось скачать. После завершения экспорта вы получаете доступ к файлам в расположении, в которое они были скачаны. Дополнительные сведения об отчетах, включаемых при загрузке результатов поиска контента, см. в разделе [Дополнительные сведения](/microsoft-365/compliance/export-search-results#more-information) в статье "Экспорт результатов поиска контента". 

После экспорта данных результаты поиска и отчеты об экспорте располагаются в папке с тем же именем, что и дело DSR. PST-файлы, содержащие элементы почтового ящика, находятся во вложенной папке с именем **Exchange**. Документы и другие элементы с сайтов находятся во вложенной папке с именем **SharePoint**.
  
## <a name="optional-step-5-revise-the-built-in-search-query"></a>(Необязательно) Шаг 5. Пересмотрите встроенный поисковый запрос

После запуска встроенного поиска вы можете отредактировать его, чтобы сузить область поиска и получить меньший объем результатов поиска. Это можно сделать, добавив в запрос условия. Условия логически связаны с запросом по ключевому слову с помощью оператора **AND**. Это означает, что элементы, чтобы попасть в результаты поиска, должны соответствовать как запросу по ключевому слову, так и всем добавленным условиям. Таким образом условия помогают сузить область результатов поиска. При добавлении двух или более уникальных (определяющих различные свойства) условий к поисковому запросу эти условия логически соединяются оператором **AND**. Это означает, что возвращаются только те элементы, которые удовлетворяют всем условиям (а кроме того, содержат ключевое слово). При добавлении нескольких значений (разделенных запятыми или точками с запятой) к одному условию эти значения соединяются оператором **OR**. Это означает, что элементы возвращаются, если они содержат любое из значений, указанных для свойства в условии.
  
Вот несколько примеров условий, которые можно добавить во встроенный поисковый запрос дела DSR. Имя фактического свойства, используемого в поисковом запросе, дано в скобках.
  
- **Тип файла (`filetype`)** – задает расширение документа или файла.. Это условие используется для поиска документов и файлов, созданных определенными приложениями Office, такими как Word, Excel или OneNote. 

- **Тип сообщения (`kind`)** — указывает тип элемента электронной почты для поиска. Например, команда `kind:email OR kind:im` вернет только сообщения электронной почты и беседы через Skype для бизнеса или приватные чаты в Microsoft Teams.

- **Тег соответствия требованиям (`compliancetag`)** — указывает метку, назначенную сообщению электронной почты или документу. Это условие возвращает элементы, классифицированные с определенной меткой. Метки используются при классификации сообщений и документов для управления данными и применения правил хранения на основе классификации. Это полезное условие для расследований DSR, поскольку ваша организация может использовать метки для классификации содержимого, связанного с конфиденциальностью данных или содержащего личные данные или конфиденциальную информацию. Для значения этого условия используйте полное имя метки или первую часть имени метки с подстановочным знаком. Дополнительные сведения см. в статье [Сведения о политиках и метках хранения](/microsoft-365/compliance/retention).

Список и описание условий, которые можно использовать в поисковом запросе, см. в разделе [Условия поиска](/microsoft-365/compliance/keyword-queries-and-search-conditions#search-conditions) статьи "Запросы по ключевым словам и условия поиска контента". 
  
### <a name="changing-the-content-locations-that-are-searched"></a>Изменение местоположений контента, в которых ведется поиск

Помимо изменения встроенного поиска для дела DSR, вы также можете изменить расположения контента, в которых ведется поиск. Как объяснялось ранее, встроенный поиск ищет во всех почтовых ящиках и на всех сайтах организации, а также в общедоступных папках Exchange Online. Например, вы можете сузить поиск, чтобы искать только в почтовом ящике субъекта данных, в его учетной записи OneDrive и на выбранных сайтах SharePoint. Если вы выбрали поиск на определенных сайтах, необходимо указать каждый сайт.
  
Чтобы изменить расположения контента для поиска, выполните следующие шаги.
  
1. Откройте встроенный поиск, для которого хотите изменить расположения контента.

2. В поисковом запросе в поле **Расположения** щелкните **Изменить** рядом с параметром **Определенные расположения** . 

    ![Нажмите кнопку "Изменить", чтобы изменить расположения контента для встроенного поискового запроса](../media/d66f7ba7-b71f-4ff5-a030-460ff02e3123.png)
  
    Появится всплывающая страница **Изменение расположений**. Далее описаны расположения контента во встроенном поиске и некоторые сведения об изменении расположений, в которых ведется поиск. 

    ![Всплывающая страница "Изменение расположений"](../media/56c033f6-6735-46ba-abb2-a263a2b79836.png)
  
    а. В разделе почтовых ящиков в верхней части всплывающей страницы установлен флажок **Выбрать все**, указывающий, что поиск должен идти во всех почтовых ящиках. Чтобы сузить область поиска, щелкните флажок, чтобы снять его, а затем нажмите кнопку **Выберите пользователей, группы или команды** и выберите определенные почтовые ящики для поиска.

    б. В разделе сайтов в верхней части всплывающей страницы установлен флажок **Выбрать все**, указывающий, что поиск должен идти на всех сайтах. Чтобы сузить поиск до выбранных сайтов, снимите этот флажок и щелкните **Выбрать сайты**. Необходимо добавить каждый сайт, на котором вы хотите проводить поиск, в том числе учетную запись OneDrive, принадлежащую искомому субъекту данных.

    в. В разделе "Общедоступные папки Exchange" установлен флажок. Это означает, что поиск ведется во всех общедоступных папках Exchange. Вы можете искать только во всех общедоступных папках Exchange либо ни в одной из них. Выбрать для поиска конкретные папки нельзя.

3. Если вы изменили расположения контента во встроенной области поиска, нажмите **Сохранить&amp; выполнить**, чтобы перезапустить поиск. 

> [!NOTE]
> Если поиск выполняется во всех или только в определенных почтовых ящиках, данные из других приложений Office 365, сохраненные в почтовых ящиках пользователей, включаются при экспорте результатов поиска. Эти данные не включаются в предполагаемые результаты поиска и недоступны для предварительного просмотра. Однако они включаются при экспорте и скачивании результатов поиска. Дополнительные сведения о приложениях, которые хранят данные в почтовом ящике пользователя, см. в статье [Контент, хранящийся в почтовых ящиках Exchange Online](/microsoft-365/compliance/what-is-stored-in-exo-mailbox).

### <a name="exporting-data-from-the-office-roaming-service"></a>Экспорт данных из службы роуминга Office

С помощью средства управления делами DSR можно искать и экспортировать данные об использовании, созданные службой роуминга Office. Роуминг Office — это служба, в которой хранятся параметры, связанные с Office, например тема Office, настраиваемый словарь, параметры языка, параметры режима разработчика и автозамены.

Данные из службы роуминга Office хранятся в почтовом ящике субъекта данных в скрытой папке, расположенной в поддереве сообщений, не входящих в IPM, почтовых ящиков Exchange Online. Это означает, что данные скрыты от пользователя, когда тот использует Outlook или другие почтовые клиенты для доступа к своему почтовому ящику. Дополнительные сведения о скрытых папках см. в статье [Скрытые папки MAPI](https://go.microsoft.com/fwlink/?linkid=872758).
  
Вы можете создать (и связать с делом DSR) отдельный поиск контента, возвращающий данные об использовании службы роуминга Office из почтового ящика субъекта данных. Эти данные не включены в статистику поиска и не будут доступны для предварительного просмотра. Но вы можете экспортировать их и предоставить субъекту данных в ответ на запрос DSR на экспорт.
  
При экспорте данных из службы роуминга Office данные сохраняются в отдельной папке, расположенной в папке **ApplicationDataRoot**, которая в свою очередь находится в папке, именем которой является адрес электронной почты субъекта данных. Эти данные экспортируются в виде JSON-файлов - текстовых файлов, понятных для человека, наподобие XML- или TXT-файлов, вложенных в сообщения электронной почты. В настоящее время эта папка использует в качестве имени глобальный уникальный идентификатор (GUID): **1caee58f-eb14-4a6b-9339-1fe2ddf6692b**. В будущих версиях средства управления делами DSR этот GUID будет заменен именем фактического приложения.

3. На странице **Присвоение имени поиску** щелкните **Отмена**. 

4. В разделе **Поисковый запрос**, в условии **Тип**, установите флажок рядом с элементом **Служба роуминга Office**.

    Условие **Тип** (которое указывает на классы сообщений электронной почты) должно быть единственным элементом в поисковом запросе. Вы можете удалить поле **Ключевые слова** или оставить его пустым. 

5. В параметре **Расположения** убедитесь, что выбрано значение **Конкретные расположения**, а затем нажмите кнопку **Изменить**.

11. Нажмите кнопку **Обзор** и укажите расположение, в которое вы хотите скачать файлы результатов поиска. 

    > [!NOTE]
    > В связи с высокой активностью диска (большим количеством операций чтения и записи) результаты поиска следует скачать на локальный диск; не скачивайте их на сетевой диск или в другое сетевое расположение. 
  
12. Нажмите кнопку **Пуск**, чтобы скачать результаты поиска на свой компьютер. 

    **Средство экспорта службы обнаружения электронных данных** отображает сведения о состоянии процесса экспорта, в том числе о предполагаемом количестве (и размере) элементов, которые осталось скачать. После завершения экспорта можно открыть PST-файл Exchange в Outlook, а затем перейти в папку **ApplicationDataRoot**, чтобы получить доступ к вложенной папке с данными службы роуминга. 

    Как объяснялось ранее, JSON-файлы, содержащие данные об использовании, вложены в сообщения. Чтобы просмотреть JSON-файл, щелкните сообщение и откройте вложенный JSON-файл.
  
### <a name="exporting-partially-indexed-items"></a>Экспорт частично индексированных элементов

Не рекомендуется экспортировать частично индексированные элементы (также называемые неиндексированными элементами) из встроенного поиска, созданного при создании дела DSR. Это потому, что результаты поиска, скорее всего, будут включать частично индексированные элементы для других пользователей в организации, а не только частично индексированные элементы для интересующего вас субъекта данных). Вместо этого рекомендуется создать отдельный поиск контента, связанный с делом DSR, предназначенный для экспорта только частично индексированных элементов, связанных с интересующим вас субъектом данных.
  
Ниже представлено общее описание экспорта частично индексированных элементов. После экспорта можно просмотреть их, чтобы определить, отвечает ли элемент условиям запроса DSR на доступ или экспорт.
  
1. Откройте дело DSR и создайте поиск на странице **Поиск**.

2. Используйте следующие условия для настройки поискового запроса и расположений контента для поиска:

    - Оставьте пустым поле для ключевого слова запроса. Такой запрос вернет все расположения контента, в которых выполняется поиск.

    - Ищите только в почтовом ящике Exchange Online субъекта данных и его учетной записи OneDrive.

3. После запуска поиска и его завершения вы можете экспортировать и скачать результаты поиска (как описано на [шаге 4](#step-4-export-the-data)). Для экспорта частично индексированных элементов используйте следующие параметры.

    - В параметре **Варианты вывода** выберите первое значение (**Только элементы, которые имеют неизвестный формат, зашифрованы или не были проиндексированы по другим причинам**), чтобы экспортировать только частично индексированные элементы.

    - Для параметра **Экспортировать содержимое Exchange как** выберите любое значение в зависимости от ваших предпочтений.

    - При выборе параметра **Включить версии для документов SharePoint** экспортируются версии документов, если версия частично индексирована.

Дополнительные сведения о частично индексированных элементах см. в статье
  
- [Частично индексированные элементы в поиске контента](/microsoft-365/compliance/partially-indexed-items-in-content-search)

- [Экспорт частично индексированных элементов](/microsoft-365/compliance/export-search-results#exporting-partially-indexed-items)

### <a name="searching-and-exporting-data-from-microsoft-teams-and-microsoft-365-groups"></a>Поиск и экспорт данных в группах Microsoft 365 и Microsoft Teams

Беседы, входящие в список чатов в Microsoft Teams, хранятся в почтовом ящике Exchange Online пользователей, участвующих в чате. Кроме того, файлы, которыми пользователь делится в приватном чате, хранятся в учетной записи OneDrive этого пользователя. Так как встроенный поиск ищет во всех почтовых ящиках и учетных записях OneDrive в организации, чаты и документы группы (созданные или загруженные субъектом данных), которыми поделились в сеансе чата, возвращаются встроенным поиском дела DSR.
  
Беседы в канале Teams, напротив, хранятся в почтовом ящике, связанном с командой. Такие беседы, в которые участвовал субъект данных, также возвращаются встроенным поиском, так как поиск ведется во всех почтовых ящиках, связанных с Teams. Кроме того, файлы, которыми субъект данных делится в канале Teams, хранятся на сайте SharePoint команды. Файлы, созданные или добавленные субъектом данных, возвращаются встроенным поиском в деле DSR, так как сайты, связанные с Teams, включаются в поиск.
  
Аналогичным образом почтовые ящики и сайты SharePoint, соответствующие группе Microsoft 365, также включаются во встроенный поиск. Это означает, что поиск вернет сообщения электронной почты, отправленные или полученные субъектом данных, а также файлы, созданные или отправленные субъектом данных. 
  
Дополнительные сведения об использовании средства поиска контента для поиска элементов в Microsoft Teams и группах Microsoft 365 и о том, как получить список участников, см. в разделе "Поиск в Microsoft Teams и группах Microsoft 365" статьи [Поиск контента в Microsoft 365](/microsoft-365/compliance/content-search-reference#searching-microsoft-teams-and-microsoft-365-groups).
  
### <a name="searching-exchange-public-folders"></a>Поиск в общедоступных папках Exchange

Встроенный поиск в деле DSR возвращает только те сообщения электронной почты, которые субъект данных отправил в общедоступную папку со включенной поддержкой почты, или сообщения, отправленные другим человеком в общедоступную папку с копией субъекту данных. Он не возвращает сообщения, которые субъект данных опубликовал в общедоступных папках. Для поиска элементов, которые субъект данных опубликовал в общедоступных папках, можно создать отдельный поиск контента, выполняющий поиск всех элементов, опубликованных в общедоступных папках субъектом данных.
  
Вот высокоуровневое описание процесса поиска элементов, которые субъект данных опубликовал в общедоступных папках. 
  
1. Откройте дело DSR и создайте поиск на странице **Поиск**. 

2. Используйте следующие критерии для настройки поискового запроса и расположений контента для поиска:

   - В поле **Ключевые слова** используйте следующий поисковый запрос: 

     ```powershell
     itemclass:ipm.post AND "<email address of the data subject>"
     ```

   - Поиск во всех общедоступных папках

   - После запуска поиска и его завершения вы можете экспортировать и скачать результаты поиска (как описано на [шаге 4](#step-4-export-the-data)). Для экспорта частично индексированных элементов используйте следующие параметры.
