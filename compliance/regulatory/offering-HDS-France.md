# <a name="health-data-hosting-hds-france"></a><span data-ttu-id="29c67-104">Стандарт размещения медицинских данных (HDS) во Франции</span><span class="sxs-lookup"><span data-stu-id="29c67-104">Health Data Hosting (HDS) France</span></span>

## <a name="about-hds"></a><span data-ttu-id="29c67-105">Сведения о HDS</span><span class="sxs-lookup"><span data-stu-id="29c67-105">About HDS</span></span>

<span data-ttu-id="29c67-106">Сертификация Hébergeurs de Données de Santé (HDS) необходима для таких организаций, как поставщики облачных служб, которые размещают на сервере персональные медицинские данные, регулируемые законодательством Франции и собранные для предоставления профилактических, диагностических и других медицинских услуг. </span><span class="sxs-lookup"><span data-stu-id="29c67-106">The Hébergeurs de Données de Santé (HDS) certification is required for entities such as cloud service providers that host the personal health data governed by French laws and collected for delivering preventive, diagnostic, and other health services.</span></span> <span data-ttu-id="29c67-107">Стандарт HDS был выпущен организацией [ASIP SANTÉ](https://esante.gouv.fr/) при Министерстве здравоохранения Франции, которая отвечает за продвижение электронных решений в сфере здравоохранения во Франции.</span><span class="sxs-lookup"><span data-stu-id="29c67-107">The HDS regulation was issued by [ASIP SANTÉ](https://esante.gouv.fr/) which, under the French Ministry of Health, is responsible for promoting electronically based healthcare solutions in France.</span></span>

<span data-ttu-id="29c67-108">Размещение медицинских данных регулируется во Франции Законом о здравоохранении (статья L.1111-8), согласно которому медицинские организации (больницы, фармацевтические компании, лаборатории), обрабатывающие персональные медицинские данные, должны использовать поставщика услуг с сертификацией HDS. </span><span class="sxs-lookup"><span data-stu-id="29c67-108">Hosting of health data is regulated under French law by the French Public Health Code (Article L.1111-8), which stipulates that any healthcare organization—hospitals, pharmaceutical companies, laboratories—that handles personal medical data must use a service provider that is HDS-certified.</span></span> <span data-ttu-id="29c67-109">В апреле 2018 г. вступили в силу новые статьи Закона о здравоохранении (R1111-8-8 — R1111-11), изменив процедуру аккредитации. Авторизация Министерством здравоохранения Франции была заменена на сертификацию уполномоченным органом, таким как Британский институт стандартов (BSI).</span><span class="sxs-lookup"><span data-stu-id="29c67-109">In April 2018, new Articles R1111-8-8 to R1111-11 of the Public Health Code took effect, changing the accreditation procedure from an authorization by the French Ministry of Health to certification by an authorized body such as BSI.</span></span>

<span data-ttu-id="29c67-110">Для сертификации HDS требуется, чтобы поставщики услуг реализовали меры, обеспечивающие безопасность, конфиденциальность и доступность персональных медицинских данных для пациентов.</span><span class="sxs-lookup"><span data-stu-id="29c67-110">HDS certification requires that service providers implement measures that keep personal health data secure, confidential, and accessible by patients.</span></span> <span data-ttu-id="29c67-111">К таким мерам относятся надежные процедуры проверки подлинности и авторизации, хорошо отлаженная система архивации и эффективные методы шифрования.</span><span class="sxs-lookup"><span data-stu-id="29c67-111">These measures include strong authentication and authorization procedures, robust backup systems, and powerful encryption methods.</span></span> <span data-ttu-id="29c67-112">Стандарт HDS также определяет обязательные положения, которые должны быть включены в контракты с поставщиком облачных служб.</span><span class="sxs-lookup"><span data-stu-id="29c67-112">HDS also specifies mandatory provisions that must be included in contracts with the cloud service provider.</span></span> <span data-ttu-id="29c67-113">Эти требования применимы независимо от места хранения данных.</span><span class="sxs-lookup"><span data-stu-id="29c67-113">These requirements apply no matter where the data is stored.</span></span>
