## <a name="how-does-microsoft-365-handle-remote-access-to-production-systems"></a><span data-ttu-id="f6697-117">Как Microsoft 365 обрабатывает удаленный доступ к производственным системам?</span><span class="sxs-lookup"><span data-stu-id="f6697-117">How does Microsoft 365 handle remote access to production systems?</span></span>

<span data-ttu-id="f6697-118">Компоненты системы Microsoft 365 находятся в центрах обработки данных, географически отделенных от оперативных групп.</span><span class="sxs-lookup"><span data-stu-id="f6697-118">Microsoft 365 system components are housed in datacenters geographically separated from the operations teams.</span></span> <span data-ttu-id="f6697-119">Сотрудники центра обработки данных не имеют логического доступа к системам Microsoft 365.</span><span class="sxs-lookup"><span data-stu-id="f6697-119">Datacenter personnel do not have logical access to Microsoft 365 systems.</span></span> <span data-ttu-id="f6697-120">В результате сотрудники служб Microsoft 365 управляют средой с помощью удаленного доступа.</span><span class="sxs-lookup"><span data-stu-id="f6697-120">As a result, Microsoft 365 service team personnel manage the environment through remote access.</span></span> <span data-ttu-id="f6697-121">Сотрудникам группы обслуживания, которым требуется удаленный доступ для поддержки Microsoft 365, предоставляется удаленный доступ только после утверждения авторизованного руководителя.</span><span class="sxs-lookup"><span data-stu-id="f6697-121">Service team personnel who require remote access to support Microsoft 365 are only granted remote access after approval from an authorized manager.</span></span> <span data-ttu-id="f6697-122">Во всем удаленном доступе используется совместимый TLS FIPS 140-2 для безопасных удаленных подключений.</span><span class="sxs-lookup"><span data-stu-id="f6697-122">All remote access uses FIPS 140-2 compatible TLS for secure remote connections.</span></span>
