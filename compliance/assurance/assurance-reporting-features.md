# <a name="microsoft-365-reporting-features"></a>Функции создания отчетов в Microsoft 365

Функции отчетности в Microsoft 365 предоставляют различные отчеты аудита для Azure Active Directory (Azure AD), Exchange Online, управления устройствами, надзорного обзора и предотвращения потери данных (DLP). Эти отчеты отличаются от отчетов Microsoft 365 действий.

## <a name="microsoft-365-reports-dashboard"></a>Microsoft 365 Панель мониторинга отчетов

Панель мониторинга отчетов в Центр администрирования Microsoft 365 отображает активность использования в Microsoft 365. Microsoft 365 глобального администратора или администратора Exchange Online, SharePoint Online или Skype для бизнеса могут получить детальное представление об использовании этой службы. Например, количество пользователей определенной Microsoft 365 службы, число пользователей, активировав Приложения Microsoft 365 для предприятий (ранее именуемой Office 365 профессиональный плюс), а также то, сколько почты проходит через организацию. Отчеты доступны за последние 7, 30, 90 и 180 дней.

## <a name="azure-ad-reports"></a>Отчеты Azure AD

Microsoft 365 Azure AD для проверки подлинности и управления удостоверением. Microsoft 365 администраторы используют отчеты, созданные Azure, для выявления необычных действий и несанкционированного доступа к своим данным. Вы можете использовать отчеты о доступе и использовании в Azure AD для получения видимости целостности и безопасности каталога для вашей организации. С помощью этих сведений можно определить и смягчить возможные риски безопасности.

Отчеты Azure AD можно экспортировать в Microsoft Excel и соотносить с другими данными из Microsoft 365. Например, результаты поиска журналов аудита могут дать представление о действиях на уровне доступа, проверки подлинности и приложений. Расширенные отчеты об аномалиях и использовании ресурсов доступны с Azure AD Premium. Эти расширенные отчеты помогают улучшить вашу осанку безопасности и помогают реагировать на потенциальные угрозы, применяя аналитику по доступу к устройствам и использованию приложений. Дополнительные сведения см. [в Azure Active Directory отчетов.](/azure/active-directory/reports-monitoring/overview-reports/)

## <a name="exchange-online-audit-reports"></a>Exchange Online Отчеты по аудиту

Exchange Online отчеты о аудите содержат сведения о доступе к почтовым ящикам и изменениях, внесенных администраторами Exchange Online клиента. При аудите почтовых ящиков можно использовать задачи в следующей таблице для запуска отчетов и экспорта Exchange Online журналов аудита.

> [!NOTE]
> Необходимо включить журнал аудита почтовых ящиков для каждого почтового ящика, чтобы в журнале аудита для этого почтового ящика были сохранены проверенные события. Если журнал аудита почтовых ящиков не включен для почтового ящика, события для этого почтового ящика не будут сохранены в журнале аудита и не будут отображаться в отчетах аудита почтовых ящиков. Дополнительные сведения см. в [том, как включить аудит почтовых ящиков.](https://support.office.com/article/Enable-mailbox-auditing-in-Office-365-aaca8987-5b62-458b-9882-c28476a66918)

## <a name="device-compliance-reports"></a>Отчеты о соответствии устройств

Вы управляете и обеспечиваете безопасность мобильных устройств, подключенных к подписке с помощью Basic Mobility and Security для Microsoft 365. Мобильные устройства, используемые для доступа к работе электронной почты, календаря, контактов и документов, играют важную роль в том, чтобы сотрудники могли работать в любое время и из любого места. Важно защитить сведения организации. Для набора политик безопасности устройств и правил доступа Microsoft 365 basic Mobility and Security. В случае потери или кражи для стирки мобильных устройств Microsoft 365 basic Mobility and Security.

В отчетах о соответствии требованиям basic Mobility and Security представлен обзор политик, установленных организацией для защиты мобильных устройств с доступом Microsoft 365 данным. Отчет позволяет фильтровать устройства по статусу соответствия требованиям, сообщениям о нарушениях, заблокированным устройствам и количеством устройств, уничтоженных в результате политики безопасности. Дополнительные сведения см. в [обзоре базовой мобильности](https://support.microsoft.com/office/overview-of-basic-mobility-and-security-for-microsoft-365-faa7d8e5-645d-4d59-839c-c8d4c1869e4a)и безопасности для Microsoft 365.

## <a name="data-loss-prevention"></a>Защита от потери данных

Политики DLP помогают управлять безопасностью и потоком информации в организации. Можно настроить политики для блокировки доступа к содержимому, шифрования данных или уведомления пользователей о нарушениях политики и политики с помощью политики DLP-Советы. Отчеты DLP дают представление о количестве совпадений политик и правил, переопределе и ложных срабатывавах.

Вы используете Центр администрирования Microsoft 365 для просмотра сведений о количестве сообщений, обнаруженных политиками DLP. Отчеты DLP предоставляют сведения о совпадениях политик и правил для отправленной и полученной почты. Вы также можете просмотреть количество совпадений, переопределей и ложных срабатывай для каждой политики в течение последних 24 часов с помощью центра администрирования Exchange. Если вы загрузите отчет Excel, вы можете просмотреть еще больше деталей, например, кто отправил сообщение, в какой день и какие совпадения политики были срабатывными. Дополнительные сведения см. в [обзоре отчетов об обнаружении политик DLP.](/previous-versions/exchange-server/exchange-150/jj889415(v=exchg.150))

## <a name="auditing-in-yammer-enterprise"></a>Аудит в Yammer корпоративный

Yammer корпоративный предоставляет администраторам возможность экспортировать данные о активности пользователей из Yammer сети через [API](https://support.office.com/article/export-data-from-yammer-enterprise-b303d8f3-007d-4ad4-81f8-54fb1ecfb3f2)экспорта Yammer данных или вручную через страницу администрирования Yammer сети. Возможность экспорта журналов ограничена сетевыми администраторами в Yammer. (Все Microsoft 365 администраторы являются Yammer сетевыми администраторами.)

Экспортируются следующие данные:

| Filename | Описание |
|----------------------------|-------------------------------------------------------------------------|
| Users.csv | Все новые, ожидающих и приостановленных пользователей в сети |
| Messages.csv | Все сообщения в сети |
| Files.csv (метаданные) | Метаданные, такие как имя файла, URL-адрес API файла, ID загрузщика, загруженные на сайте и т. д. |
| Files.csv (исходные файлы) | Почтовый файл исходных файлов, загруженных пользователями в Yammer |
| Topics.csv | Темы, созданные в сети |
| Pages.csv | Страницы (заметки), созданные пользователями в сети |
| Admins.csv | Все проверенные администраторы в сети |
| Networks.csv | Все Yammer внешние сети |

Yammer корпоративный также доступны в отчетах Microsoft 365 действий. Кроме того, Yammer активно работает над разоблачение дополнительных журналов через API Microsoft 365 управления и над возможностью аргументировать данные с помощью Power BI. Дополнительные сведения об этих Office см. в Office [roadmap.](https://fasttrack.microsoft.com/roadmap?filters=yammer)
