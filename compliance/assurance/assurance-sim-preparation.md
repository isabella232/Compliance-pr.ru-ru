# <a name="microsoft-365-security-incident-management-preparation"></a>Управление инцидентами безопасности Microsoft 365: подготовка

## <a name="training-and-background-checks"></a>Обучение и проверка фона

Каждый сотрудник, работающий в Microsoft 365, предоставляется обучение инцидентам безопасности и процедурам реагирования, соответствующим их роли. Каждый сотрудник Корпорации Майкрософт 365 проходит обучение по присоединению и ежегодное обучение по переподготовке каждый год после этого. Обучение предназначено для того, чтобы предоставить сотруднику базовое представление о подходе Корпорации Майкрософт к безопасности, чтобы по завершении обучения все сотрудники понимали:

- Определение инцидента с безопасностью
- Ответственность всех сотрудников сообщать об инцидентах безопасности
- Эскалация потенциального инцидента с безопасностью в группе реагирования на безопасность Microsoft 365
- Как группа реагирования на инциденты безопасности Microsoft 365 реагирует на инциденты безопасности
- Особые проблемы, касающиеся конфиденциальности, особенно конфиденциальности клиентов
- Где найти дополнительные сведения о безопасности и конфиденциальности и контактах эскалации
- Любые другие соответствующие области безопасности (по мере необходимости)

Соответствующие сотрудники ежегодно получают курсы повышения квалификации по вопросам безопасности. В ежегодном обучении по обновлению основное внимание уделяется:

- Любые изменения, внесенные в стандартные операционные процедуры в предыдущем году
- Ответственность каждого человека за сообщение об инцидентах с безопасностью, а также о том, как это сделать
- Где найти дополнительные сведения о безопасности и конфиденциальности и контактах эскалации
- Любые другие области фокуса безопасности, которые могут иметь значение каждый год

Каждый сотрудник, работающий в Microsoft 365, также проходит соответствующую и тщательную проверку, которая включает образование, занятость кандидата, криминальную историю и другую конкретную информацию по нормативным актам США, таким как Закон о переносе и подотчетности медицинского страхования (HIPAA), Международный трафик в правилах вооружений (ITAR), Федеральная программа управления рисками и авторизациями (FedRAMP) и другие.

Проверка фона является обязательной для всех сотрудников, работающих в инженерной службе Microsoft 365. Некоторые среды Microsoft 365 и роли оператора также могут требовать полной дактилоскопии, требований к гражданству, требований к разрешению правительства и других более строгих элементов управления. Кроме того, некоторые группы служб и роли при необходимости могут проходить специальную подготовку по вопросам безопасности. Наконец, члены группы безопасности сами получают специализированную подготовку и участие в конференциях, непосредственно связанных с безопасностью. Это обучение зависит от потребности команды и сотрудника, но включает в себя такие вещи, как отраслевые конференции, внутренние конференции Microsoft Security и внешние учебные курсы через известных поставщиков подготовки по безопасности в отрасли. Мы также регулярно публикуем статьи по обучению безопасности, опубликованные в течение года для сообщества безопасности в Корпорации Майкрософт и регулярно специализируются на Microsoft 365.

## <a name="penetration-testing--assessment"></a>Оценка & проверки & проникновения

Корпорация Майкрософт работает с различными отраслевыми органами и экспертами по безопасности для понимания новых угроз и развивающихся тенденций. Корпорация Майкрософт непрерывно оценивает собственные системы на наличие уязвимостей и заключает контракты с различными независимыми внешними экспертами, которые делают то же самое.

Тесты, проведенные для затвердения службы в Microsoft 365, можно сгруппить по четырем общим категориям:

1. **Автоматическое тестирование безопасности:** Внутренние и внешние сотрудники регулярно сканируют среду Microsoft 365, основанную на практиках Microsoft SDL, Open Web Application Security Project (OWASP) Top 10 рисков и возникающих угроз, о чем сообщают различные отраслевые органы.
2. **Оценки уязвимости:** Формальные взаимодействия с независимыми сторонними тестерами регулярно проверяют, эффективно ли работают ключевые логические элементы управления для выполнения обязательств по обслуживанию различных контролирующих органов. Оценки проводятся сертифицированными сотрудниками Совета зарегистрированных тестеров этической безопасности (CREST) и основаны на рисках OWASP Top 10 и других угрозах, применимых к службе. Все найденные угрозы отслеживаются до закрытия.
3. **Непрерывное тестирование уязвимости системы:** Корпорация Майкрософт проводит регулярное тестирование, в ходе которого группы пытаются нарушить систему с помощью возникающих угроз, смешанных угроз и/или расширенных постоянных угроз, в то время как другие группы пытаются заблокировать такие попытки взлома.
4. **Microsoft Online Services программы bug Bounty.** В этой программе действует политика допуска ограниченных, исходя из клиентов, оценок уязвимости в Microsoft 365. Дополнительные сведения см. [в Microsoft Online Services Bug Bounty Terms.](https://www.microsoft.com/msrc/bounty-terms)

Команда инженеров Microsoft 365 периодически публикует различные документы соответствия требованиям. Некоторые из этих документов доступны в соглашении о неразглашении на портале доверия [microsoft Cloud Service или](https://aka.ms/STP) в области обеспечения безопасности служб Центра соответствия требованиям Microsoft [365.](https://compliance.office.com)

>[!NOTE]
>Подробнее о том, как получить доступ к порталу доверия к службе, см. в материале "Начало работы с порталом доверия служб для Office 365 для бизнеса, подписки Azure и Dynamics CRM Online". Для доступа к центру соответствия требованиям Microsoft 365 требуется подписка Microsoft 365.

## <a name="attack-simulation"></a>Имитация атаки

Корпорация Майкрософт участвует в текущих упражнениях моделирования атак и тестировании на проникновение на веб-сайте наших планов безопасности и ответов с целью улучшения возможностей обнаружения и реагирования. Корпорация Майкрософт регулярно имитирует реальные нарушения, проводит непрерывный мониторинг безопасности и практикует реагирование на инциденты безопасности для проверки и повышения безопасности Microsoft 365 и Azure.

Корпорация Майкрософт выполняет предполагаемую стратегию безопасности нарушений с помощью двух основных групп:

### <a name="red-teams"></a>Красные команды

Красная группа Безопасности Microsoft 365 — это группа штатных сотрудников Корпорации Майкрософт, которая фокусируется на нарушении инфраструктуры, платформы и собственных клиентов и приложений Майкрософт. Это выделенный противник (группа этических хакеров), который выполняет целенаправленные и постоянные атаки на веб-службы (но не на приложения или данные клиентов). Они обеспечивают непрерывную проверку "полного спектра" (например, технические элементы управления, бумажная политика, реакция человека и т. д.) возможностей реагирования на инциденты службы.

### <a name="blue-teams"></a>Синие команды

Команда Microsoft 365 Security Blue Team состоит из специального набора служб реагирования на безопасность и участников из различных групп реагирования на инциденты безопасности, инженерных и оперативных групп. Они независимы и работают отдельно от красной команды. Blue Team следует установленным процессам безопасности и использует новейшие средства и технологии для обнаружения и реагирования на атаки и попытки проникновения. Так же, как атаки в реальном мире, blue Team не знает, когда и как будут происходить атаки красной команды или какие методы могут быть использованы. Их задача, будь то атака Red Team или фактические атаки, заключается в обнаружении и реагировании на все инциденты безопасности. По этой причине blue Team постоянно находится на вызове и должна реагировать на нарушения Red Team так же, как и для любого другого противника.

Сотрудники Корпорации Майкрософт разделяют красные и синие команды в различных подразделениях, которые проводят операции как между службами, так и внутри них. Этот подход, именуемый *red Teaming,* заключается в проверке систем и операций служб Майкрософт с использованием той же тактики, методов и процедур, что и реальные противники, в отношении инфраструктуры живого производства, без foreknowledge инженерных и операционных групп инфраструктуры и платформы. Это проверяет возможности обнаружения безопасности и реагирования, а также позволяет контролировать обнаружение производственных уязвимостей, ошибок конфигурации, недействительных предположений или других проблем безопасности. Каждое нарушение Red Team сопровождается полным раскрытием информации между красной командой и синей командой, включая службы, для выявления пробелов, устранения выводов и существенного улучшения реагирования на нарушение.

>[!NOTE]
>Данные клиентов не ориентированы во время red Teaming или в прямом эфире упражнений по проникновению сайта. Тесты проводятся с инфраструктурой и платформами Microsoft 365 и Azure, а также с собственными клиентами, приложениями и данными Майкрософт. Клиенты, приложения и данные, которые находятся в Microsoft 365 или Azure, никогда не являются целевыми по согласованным правилам взаимодействия.

### <a name="joint-exercises"></a>Совместные упражнения

Иногда команды Microsoft 365 Security Blue и Red будут проводить совместные операции, в которых отношения во время операции являются более партнерскими, чем состязательной, с выбранным набором сотрудников из каждой группы. Эти упражнения хорошо согласованы между группами для получения более целевого набора результатов с помощью совместной работы этических хакеров и ответчиков в режиме реального времени. Эти упражнения "фиолетовой команды" очень адаптированы для каждой операции, чтобы максимально использовать эту возможность, но для каждой операции важное значение имеет обмен информацией с высокой пропускной способностью и партнерство для достижения поставленных целей.
