<span data-ttu-id="c14b9-114">Exchange Online, Microsoft Teams, SharePoint Online и OneDrive для бизнеса также используют шифрование служб на уровне приложений для шифрования контента клиентов.</span><span class="sxs-lookup"><span data-stu-id="c14b9-114">Exchange Online, Microsoft Teams, SharePoint Online, and OneDrive for Business also use Service Encryption at the application layer to encrypt customer content.</span></span> <span data-ttu-id="c14b9-115">Шифрование служб обеспечивает защиту прав и функции управления на вершине сильной защиты шифрования.</span><span class="sxs-lookup"><span data-stu-id="c14b9-115">Service Encryption provides rights protection and management features on top of strong encryption protection.</span></span> <span data-ttu-id="c14b9-116">Она также позволяет разделять операционные системы Windows и данные клиентов, хранимые или обрабатываемых этими операционными системами.</span><span class="sxs-lookup"><span data-stu-id="c14b9-116">It also allows for separation between Windows operating systems and the customer data stored or processed by those operating systems.</span></span>

## <a name="how-does-microsoft-365-encrypt-data-in-transit"></a><span data-ttu-id="c14b9-117">Как Microsoft 365 шифрует данные при транзите?</span><span class="sxs-lookup"><span data-stu-id="c14b9-117">How does Microsoft 365 encrypt data-in-transit?</span></span>

<span data-ttu-id="c14b9-118">Продукты и службы Microsoft 365 используют прочные транспортные протоколы, такие как TLS, чтобы предотвратить прослушивание неавторизованной стороной данных клиентов во время перемещения по сети.</span><span class="sxs-lookup"><span data-stu-id="c14b9-118">Microsoft 365 products and services use strong transport protocols, such as TLS, to prevent unauthorized parties from eavesdropping on customer data while it moves over a network.</span></span> <span data-ttu-id="c14b9-119">Примеры транзитных данных включают почтовые сообщения, которые находятся в процессе доставки, беседы, происходящие в онлайн-собрании, или файлы, реплицированные между центрами обработки данных.</span><span class="sxs-lookup"><span data-stu-id="c14b9-119">Examples of data in transit include mail messages that are in the process of being delivered, conversations taking place in an online meeting, or files being replicated between datacenters.</span></span>

<span data-ttu-id="c14b9-120">В Microsoft 365 данные считаются "транзитными", когда устройство пользователя общается с сервером Майкрософт или сервером Майкрософт, который общается с другим сервером.</span><span class="sxs-lookup"><span data-stu-id="c14b9-120">In Microsoft 365, data is considered 'in transit' whenever a user's device is communicating with a Microsoft server or a Microsoft server is communicating with another server.</span></span> <span data-ttu-id="c14b9-121">Exchange Online, SharePoint Online, Microsoft Teams и Office Online используют TLS для обеспечения конфиденциальности данных во время транзита.</span><span class="sxs-lookup"><span data-stu-id="c14b9-121">Exchange Online, SharePoint Online, Microsoft Teams, and Office Online all use TLS to ensure data remains confidential while in transit.</span></span>

## <a name="how-does-microsoft-365-manage-the-keys-used-for-encryption"></a><span data-ttu-id="c14b9-122">Как Microsoft 365 управляет ключами, используемыми для шифрования?</span><span class="sxs-lookup"><span data-stu-id="c14b9-122">How does Microsoft 365 manage the keys used for encryption?</span></span>

<span data-ttu-id="c14b9-123">Надежное шифрование является безопасным только для ключей, используемых для шифрования данных.</span><span class="sxs-lookup"><span data-stu-id="c14b9-123">Strong encryption is only as secure as the keys used to encrypt data.</span></span> <span data-ttu-id="c14b9-124">Корпорация Майкрософт использует собственные сертификаты безопасности для шифрования подключений TLS для передачи данных.</span><span class="sxs-lookup"><span data-stu-id="c14b9-124">Microsoft uses its own security certificates to encrypt TLS connections for data-in-transit.</span></span> <span data-ttu-id="c14b9-125">Для данных на отдыхе объемы, защищенные BitLocker, шифруются с ключом шифрования полного объема, который шифруется ключом магистрали тома, который, в свою очередь, связан с доверенным модулем платформы (TPM) на сервере.</span><span class="sxs-lookup"><span data-stu-id="c14b9-125">For data-at-rest, BitLocker-protected volumes are encrypted with a full volume encryption key, which is encrypted with a volume master key, which in turn is bound to the Trusted Platform Module (TPM) in the server.</span></span> <span data-ttu-id="c14b9-126">BitLocker использует алгоритмы, совместимые с FIPS, чтобы гарантировать, что ключи шифрования никогда не хранятся и не отправляются по проводу в чистом.</span><span class="sxs-lookup"><span data-stu-id="c14b9-126">BitLocker uses FIPS-compliant algorithms to ensure that encryption keys are never stored or sent over the wire in the clear.</span></span>

<span data-ttu-id="c14b9-127">Шифрование служб обеспечивает дополнительный уровень шифрования данных клиентов в Exchange Online, Microsoft Teams, SharePoint Online и OneDrive для бизнеса.</span><span class="sxs-lookup"><span data-stu-id="c14b9-127">Service Encryption provides an additional layer of encryption for customer data-at-rest in Exchange Online, Microsoft Teams, SharePoint Online, and OneDrive for Business.</span></span> <span data-ttu-id="c14b9-128">Шифрование служб предоставляет клиентам два варианта управления ключами шифрования: клавиши с управлением Майкрософт или клиентский ключ.</span><span class="sxs-lookup"><span data-stu-id="c14b9-128">Service Encryption gives customers two options for encryption key management: Microsoft-managed keys or Customer Key.</span></span> <span data-ttu-id="c14b9-129">При использовании ключей под управлением Майкрософт службы Microsoft 365 автоматически создают и надежно хранят корневые ключи, используемые для шифрования служб.</span><span class="sxs-lookup"><span data-stu-id="c14b9-129">When using Microsoft-managed keys, Microsoft 365 services automatically generate and securely store the root keys used for Service Encryption.</span></span>

<span data-ttu-id="c14b9-130">Клиенты с требованиями для управления собственными ключами корневого шифрования могут использовать шифрование служб с помощью ключа клиента.</span><span class="sxs-lookup"><span data-stu-id="c14b9-130">Customers with requirements to control their own root encryption keys can leverage Service Encryption with Customer Key.</span></span> <span data-ttu-id="c14b9-131">С помощью ключа клиента клиенты могут создавать собственные криптографические ключи с помощью локального модуля службы оборудования (HSM) или хранилища ключей Azure (AKV).</span><span class="sxs-lookup"><span data-stu-id="c14b9-131">Using Customer Key, customers can generate their own cryptographic keys using either an on-premises Hardware Service Module (HSM) or Azure Key Vault (AKV).</span></span> <span data-ttu-id="c14b9-132">Корневые ключи клиента хранятся в AKV, где они могут использоваться в качестве корневого ключа одного из ключей, шифруемого данными или файлами почтовых ящиков клиентов.</span><span class="sxs-lookup"><span data-stu-id="c14b9-132">Customer root keys are stored in AKV, where they can be used as the root of one of the keychains that encrypts customer mailbox data or files.</span></span> <span data-ttu-id="c14b9-133">Корневые ключи клиента могут быть доступны только косвенно кодом службы Microsoft 365 для шифрования данных и не могут быть доступны непосредственно сотрудниками Майкрософт.</span><span class="sxs-lookup"><span data-stu-id="c14b9-133">Customer root keys can only be accessed indirectly by Microsoft 365 service code for data encryption and cannot be accessed directly by Microsoft employees.</span></span>
