# <a name="governance-overview"></a>Обзор управления

## <a name="how-does-microsoft-provide-effective-security-governance-across-the-enterprise"></a>Как корпорация Майкрософт обеспечивает эффективное управление безопасностью на предприятии?

Корпорация Майкрософт понимает, что эффективные политики безопасности должны последовательно внедряться на предприятии для защиты информационных систем и клиентов Корпорации Майкрософт. Политики безопасности также должны учитывать универсальное применение различных функций бизнеса и информационных систем. Чтобы соответствовать этим требованиям, Корпорация Майкрософт реализует всеобъемлющую программу управления безопасностью в рамках Microsoft Policy Framework. Управление безопасностью подпадает под политику безопасности Майкрософт (MSP).

MSP организует политики безопасности, стандарты и требования Корпорации Майкрософт, чтобы они могли быть реализованы во всех инженерных группах и бизнес-подразделениях Майкрософт. Отдельные бизнес-подразделения отвечают за конкретные реализации политик безопасности Майкрософт. Например, Microsoft 365 реализации своей безопасности в Microsoft 365 политики информационной безопасности и связанной Microsoft 365 Control Framework. Azure и Dynamics 365 документировать свои реализации безопасности в стандартных операционных процедурах (SOP) и Azure Control Framework. Эти реализации безопасности соответствуют целям и задачам MSP.

Программа управления безопасностью Корпорации Майкрософт информируется и согласуется с различными нормативными и нормативными рамками. Требования безопасности постоянно развиваются, чтобы учитывать новые технологии, требования к регулированию и соответствие требованиям, а также угрозы безопасности. В связи с этими изменениями Корпорация Майкрософт регулярно обновляет политики безопасности и вспомогательные документы для защиты систем и клиентов Майкрософт, выполнения наших обязательств и поддержания доверия клиентов.

## <a name="how-do-microsoft-online-services-implement-the-microsoft-security-policy-msp"></a>Как интернет-службы Майкрософт реализуют политику безопасности Майкрософт (MSP)?

Microsoft 365 документы реализации безопасности в Microsoft 365 политики информационной безопасности. Эта политика соответствует политике безопасности Майкрософт и регулирует информационную систему Microsoft 365, включая все Microsoft 365 среды и все ресурсы, связанные с коллекцией, обработкой, обслуживанием, использованием, обменом, распространением и удалением данных. Кроме того, Azure и Dynamics 365 используют политику безопасности Майкрософт для управления своей информационной системой.

Информационные системы включают следующие компоненты, Microsoft 365 политики информационной безопасности (для Microsoft 365) и Политики безопасности Microsoft (для Azure и Dynamics 365):

- Инфраструктура: физические и аппаратные компоненты Azure, Dynamics 365 и Microsoft 365 (средства, оборудование и сети)
- Программное обеспечение: программы и операционное программное обеспечение Azure, Dynamics 365 и Microsoft 365 (системы, приложения и утилиты)
- Люди. Персонал, участвующий в работе и использовании систем Azure, Dynamics 365 и Microsoft 365 (разработчики, операторы, пользователи и менеджеры)
- Процедуры: запрограммированные и ручные процедуры, участвующие в работе систем Azure, Dynamics 365 и Microsoft 365.
- Данные: сведения, созданные, собранные и обработанные системами Azure, Dynamics 365 и Microsoft 365 (потоки транзакций, файлы, базы данных и таблицы)

Политика Microsoft 365 безопасности информации дополняется рамочной Microsoft 365 управления. В Microsoft 365 Control Framework подробные сведения о требованиях к минимальной безопасности для всех Microsoft 365 и компонентов информационной системы. В нем также ссылаются на юридические и корпоративные требования, предъявляемые к каждому контролю. Эта структура включает имена, описания и инструкции по контролю для обеспечения эффективной реализации команд служб. Microsoft 365 для отслеживания реализации управления для внутренней и внешней отчетности. Аналогичным образом в Azure и Dynamics 365 реализации управления записями в Azure Control Framework.

## <a name="how-do-online-services-limit-and-track-exceptions-to-established-policies-and-procedures"></a>Как сетевые службы ограничивают и отслеживают исключения для установленных политик и процедур?

Все исключения из рамок управления должны иметь законное обоснование бизнеса и быть утверждены соответствующей организацией управления в каждой группе сетевых служб. В зависимости от объема исключения и потенциального риска, который он представляет, может потребоваться утверждение исключений от корпоративного вице-президента или более высокого уровня. Исключения управляются в средстве отслеживания, где они рассматриваются и утверждены для сохраняемой актуальности.

## <a name="how-do-online-services-keep-security-and-compliance-requirements-updated"></a>Как онлайн-службы обновляют требования к безопасности и требованиям соответствия требованиям?

Группы управления, риска и соответствия требованиям каждой онлайн-службы (GRC) работают для поддержания системы управления на постоянной основе. В нескольких сценариях может потребоваться, чтобы группа GRC обновила структуру управления, включая изменения в соответствующих нормативных актах или законах, возникающие угрозы, результаты тестирования на проникновение, инциденты с безопасностью, отзывы аудита и новые требования к соответствия требованиям. Когда требуется изменение структуры, группа Trust определяет ключевых заинтересованных лиц, ответственных за одобрение и реализацию этого изменения, чтобы убедиться, что оно возможно и не вызовет непредвиденных проблем с онлайн-службами. После того как группа GRC и соответствующие заинтересованные стороны договорятся о том, что требуется для изменения, рабочие нагрузки, ответственные за реализацию изменений, устанавливают целевые даты завершения и работают над реализацией изменений в соответствующих службах. После выполнения целей реализации команда Trust обновляет рамки управления с помощью новых или обновленных элементов управления.
